# Mandatory Delivery Flow (AGENTS.md Standard)

This skill automates and enforces the **7-step Mandatory Delivery Flow** defined in `AGENTS.md` for every feature, fix, or refactoring task in this repository.

---

## 📋 Delivery Flow Checklist & Command Reference

### Step 1: Task & Issue Management
1. Verify task entry exists in `TASKS.md` under `TASK INDEX` and `正式タスク詳細`.
2. Ensure status order (`in_progress` -> `done`) and Task ID descending rule.
3. Ensure corresponding GitHub Issue exists (or create one using `gh issue create`).
   ```bash
   gh issue create --title "[TASK-xxx] <Title>" --body "<Description & Acceptance Criteria>"
   ```

### Step 2: Branch Management
1. Branch must follow format: `<type>/TASK-<id>-<short-description>` (e.g., `feat/TASK-316-ui-ux-comprehensive-improvements`).
2. Verify current branch:
   ```bash
   git branch --show-current
   ```

### Step 3: Implementation & Mandatory Quality Checks
Run all 3 quality gates with zero omissions before committing:
```bash
# 1. Linter check (Must pass with 0 errors)
npm run lint

# 2. Unit / Integration tests (Must pass 100%)
npx vitest run

# 3. Playwright E2E tests (Must pass 100%)
npm run test:e2e
```
*If port conflicts occur during E2E tests, clean port 3100 via `fuser -k 3100/tcp || true` and retry.*

### Step 4: Self Review & Commit
1. Check changed files and ensure no temporary/autogenerated noise (e.g. `next-env.d.ts` dev mutations):
   ```bash
   git status
   git diff
   ```
2. Commit using Conventional Commit format:
   ```bash
   git add .
   git commit -m "<type>: TASK-<id> <short-description>"
   ```

### Step 5: Push & PR Creation
1. Push branch to remote:
   ```bash
   git push -u origin <branch-name>
   ```
2. Create Pull Request linked to the GitHub Issue:
   ```bash
   gh pr create --title "<type>: <title> [TASK-<id>]" --body "## 概要\n<Summary>\n\n## 関連Issue\nCloses #<issue_id>\n\n## チェックリスト\n- [x] npm run lint がパスすること\n- [x] npx vitest run が全件パスすること\n- [x] npm run test:e2e が全件パスすること\n- [x] 関連Issue (#<issue_id>) をクローズ対象に指定"
   ```

### Step 6: CI Confirmation (Wait for ALL GREEN)
Monitor GitHub Actions CI status until all checks pass:
```bash
gh pr checks <pr_number>
```
- Must wait until all required checks (`CI/Lint`, `CI/Unit/Integration`, `CI/DB Integration`, `CI/E2E`, `CI/PR Operational Guard`, `Vercel`) display `✓` (successful).
- If any check fails, fix the issue, commit, push, and re-check until green.

### Step 7: Merge to `main`, Sync Local & Cleanup
1. Merge PR and delete remote branch:
   ```bash
   gh pr merge <pr_number> --merge --delete-branch
   ```
2. Sync local `main` branch with latest remote:
   ```bash
   git checkout main
   git pull origin main
   ```
3. Delete local feature branch:
   ```bash
   git branch -d <branch-name>
   ```
4. Verify corresponding GitHub Issue is closed (`gh issue view <issue_id>`).
5. Update `TASKS.md` status to `done` if not updated already.

---

## ⚡ Execution Guidelines for Agent

- **Never skip quality checks**: Always run `npm run lint`, `npx vitest run`, and `npm run test:e2e`.
- **Never merge without CI green**: Check `gh pr checks` and confirm all 7+ checks pass before merging.
- **Never leave orphan branches**: Always delete local feature branch and sync local `main` after merging.
