## 1. 価格戦略

### 価格帯比較分析

| 価格 | メリット | デメリット | 推奨判定 |
|---|---|---|---|
| **480円/月** | 参入障壁が低い。ユーザー獲得しやすい | 「安すぎる」印象。品質への不信感。収益性が低い | 非推奨 |
| **680円/月** ← 現状 | 「缶コーヒー2本分」の心理的許容範囲。収益バランス良好 | 特に問題なし | **推奨** |
| **980円/月** | 高品質感。収益改善 | 無料→有料の転換率低下リスク | 段階的値上げの目標価格 |
| **1,480円/月** | プレミアム訴求。動物病院連携に適した価格 | 個人ユーザーには高い | B2B/連携版向け |
| **1,980円/月** | エンタープライズ価格 | 個人ユーザーの離脱リスク大 | 非推奨（初期） |

### 680円の価値設計

**「680円を払い続ける理由」をユーザー目線で設計**:

```
1. 保険的価値（最重要）
   「もし緊急事態が起きたとき、この情報が命を救うかもしれない」
   → 年間8,160円は、ペットの命のための保険料と考えると安い

2. 記録の蓄積価値（ロックイン）
   「3年分の医療記録がここにある。他に移れない」
   → データの蓄積が継続利用インセンティブになる

3. 利便性価値
   「毎回問診票を書かなくていい。QR一枚で済む」
   → 時間節約 × 安心感

4. 家族共有価値
   「夫も子供もアプリで確認できる。一家全員で安心」
   → 世帯単位の価値が個人課金より高い

5. 社会的責任感
   「ペットを飼う責任として、最低限の記録は残しておくべき」
   → 義務感が継続利用を支援
```

### 無料版との差別化

```
無料版（トライアル終了後）:
  - 記録の閲覧（30日以内）
  - QR公開画面の閲覧のみ

有料版（月額680円）:
  - 全機能利用
  - 記録の無制限閲覧・作成・編集
  - 家族共有（人数制限なし）
  - 複数ペット管理（頭数制限なし）
  - 投薬リマインダー
  - QRコード生成・共有
  - OCR医療書類読取
  - PDF出力
  - 変更履歴
```

### 競合価格比較

| サービス | 価格 | 対象 | 特徴 |
|---|---|---|---|
| 動物の救急手帳 | 680円/月 | ペット全般 | 緊急QR特化・家族共有 |
| ペット手帳（一般的なアプリ） | 無料〜500円 | ペット全般 | 基本記録のみ |
| ペットの医療記録アプリ（海外） | $2-5/月 | ペット全般 | 記録特化 |
| かかりつけ病院のアプリ | 無料 | 特定病院患者 | 特定病院のみ |
| ペット保険 | 2,000〜10,000円/月 | ペット全般 | 保険＋記録 |

**差別化ポイント**:
- 緊急時QRコード公開機能（他にない）
- 家族全員で共有（一般的なアプリにない）
- 医療従事者向けの閲覧最適化
- 医療情報の構造化記録

## 2. 継続利用設計

### チャーン（解約）防止設計

#### 解約率低減の主要施策

```
1. 医療記録の蓄積によるスイッチングコスト
   効果: 利用期間が長くなるほど解約しにくくなる
   実装: 記録件数・利用期間をダッシュボードに表示
         「○○との3年間の記録」という感情的な訴求

2. 年払い割引
   効果: 解約検討頻度の低下（月次 vs 年次）
   実装: 月額680円 vs 年払い6,480円（2ヶ月分お得）

3. ワクチン・投薬リマインダーへの依存
   効果: アプリが「ないと困る」存在になる
   実装: プッシュ通知・メール通知の精度向上

4. 解約阻止フロー（解約理由ヒアリング）
   効果: 解約理由から改善ポイントを特定
   実装: 解約前に解約理由を選択してもらう
         「高い」→ 価値訴求 / 「使わない」→ 機能説明

5. ペット訃報ケアの設計
   問題: ペットが亡くなった際に解約される（最大の解約トリガー）
   解決策: 「思い出として保管」機能・アーカイブモード
```

#### 解約理由別の対策

| 解約理由 | 推定比率 | 対策 |
|---|---|---|
| ペットの死亡 | 30% | アーカイブモード。「○○との思い出を残す」UX |
| 高いと感じた | 25% | 価値の再訴求。年払いオファー |
| 使わなくなった | 20% | エンゲージメント向上。通知最適化 |
| 機能不満 | 15% | フィードバック収集・改善 |
| 競合乗換 | 10% | 差別化機能の強化 |

## 3. オンボーディング設計

### 3ステップオンボーディング（TASK-166実装済み）

```
ステップ1: ペット登録
  目標: 最低限の情報でペットを登録させる
  必須: ペット名・種類
  任意: 品種・性別・誕生日（あとで追加可）
  完了後: 「チョコを登録しました！次は緊急情報を登録しましょう」

ステップ2: 緊急情報入力
  目標: 最重要の緊急情報を入力させる
  必須: 緊急連絡先電話番号
  推奨: 持病・アレルギー・かかりつけ病院
  完了後: 「あとはQRコードを確認するだけです」

ステップ3: QRコード確認
  目標: QRコードの価値を体験させる
  操作: QRコードを表示→自分でスキャンして確認
  完了後: 「設定完了！いざという時に使えます」
```

### 「30日間で価値を実感」設計

```
Day 1-3: 基本設定完了（オンボーディング）
Day 7: 投薬リマインダーを初めて受信（価値体験）
Day 14: 体重・検査値グラフで推移を確認（継続モチベーション）
Day 21: 医療記録を初めて使う（病院受診後）
Day 30: トライアル終了 → 課金転換

→ 30日で複数の「ないと困る」体験を積む設計
```

## 4. 課金導線設計

### 課金転換フロー

```
無料トライアル期間中:
  - 全機能を制限なく利用
  - Day 7: 課金の案内（ソフト）「30日後に続けますか？」
  - Day 23: 課金の案内（強め）「あと7日で機能制限されます」
  - Day 30: 課金の案内（強制）「トライアルが終了しました」

課金後:
  - Stripe Checkout でクレジットカード入力
  - 成功後: 即座に全機能アクティブ
```

### 課金ページの設計原則

```
メッセージ:
  NG: 「月額680円を支払ってください」（コスト訴求）
  OK: 「チョコの緊急情報を、いざという時のために守りませんか」（価値訴求）

表示内容:
  1. ペットの写真（感情的訴求）
  2. 現在登録されている記録件数（データの価値の可視化）
  3. 月額680円の価値比較（「缶コーヒー2本分」）
  4. 「今すぐ続ける」ボタン（Primary CTA）
  5. 「あとで（機能が制限されます）」（Secondary）

社会的証明（将来）:
  「○○人の飼い主が緊急時に使用しました」
```

## 5. LTV向上戦略

### LTV計算式

```
LTV = 平均月額 × 平均利用期間（月）
    = 680円 × 36ヶ月（3年）
    = 24,480円

目標:
  平均利用期間: 24ヶ月以上（業界平均: 12ヶ月）
  有料転換率: 20%以上（業界平均: 2-5%）
```

### LTV向上施策

```
1. 家族プラン（追加課金なし）
   現在: 世帯単位で課金（メンバー追加無料）
   改善: 家族プランの価値をより明確に訴求
   
2. 複数ペット割引（将来）
   4頭以上: 月額980円（1頭あたり245円）
   → 多頭飼い家庭の継続利用を促進

3. 紹介プログラム
   紹介された人: 30日→60日の延長トライアル
   紹介した人: 1ヶ月分の割引
   → バイラル成長 + 既存ユーザーのエンゲージメント

4. 動物病院連携（将来・B2B）
   動物病院が加盟: ペット情報を受け取れる
   飼い主側: かかりつけ病院と直接連携できる
   → ネットワーク効果・スイッチングコスト向上

5. ペット保険会社連携（将来・B2B）
   保険会社が利用: 請求時の記録確認
   飼い主側: 保険申請が自動化される
   → 価値向上・新収益源
```

### CAC削減戦略

```
有機的成長（メインチャネル）:
  - SEO: 「ペット 緊急時 情報」「犬 救急 アプリ」等の検索流入
  - 口コミ: 紹介プログラム + 緊急時に使用した体験談
  - SNS: Twitterでの飼い主コミュニティへの訴求

有料獲得:
  - Google/Yahoo検索広告（ペット・救急関連キーワード）
  - Instagram広告（ペット写真での感情訴求）
  - 動物病院への提携営業

想定CAC:
  目標: < 3,000円（LTV 24,480円に対して12%）
  初期: 5,000〜10,000円（ブランド認知がない間）
```

## 6. 管理画面設計

### 管理画面（未実装・設計）

#### 必要な管理画面

```
ダッシュボード:
  - アクティブユーザー数（DAU/MAU）
  - 有料ユーザー数・転換率
  - 売上・MRR・ARR
  - 解約数・解約率

ユーザー管理:
  - ユーザー一覧（email, 登録日, 課金状態）
  - 特定ユーザーのサブスク状態確認・変更
  - ユーザー検索・フィルター

課金管理:
  - サブスクリプション一覧
  - 支払い失敗の確認
  - 返金処理（Stripe Portal連携）

問い合わせ管理:
  - 問い合わせ一覧（タイプ・ステータス）
  - 担当者アサイン・返答

監査ログ:
  - セキュリティイベントの一覧
  - 特定ユーザーの行動履歴
  - 異常検知アラート

お知らせ・FAQ管理:
  - アプリ内お知らせの作成・配信
  - FAQの追加・編集
  - バナー管理

障害管理:
  - エラー件数・傾向
  - ステータスページとの連携
```

#### 管理画面の認証

```
要件:
  - 通常のSupabase Auth とは別の管理者認証
  - 多要素認証（MFA）必須
  - IP制限（オフィス・VPN のみ）
  - セッションタイムアウト: 30分
  - 全操作の監査ログ記録
```

## 7. KPI設計

### 主要KPI

```
獲得:
  - CAC（顧客獲得コスト）: 目標 < 3,000円
  - 新規登録数（週次）
  - 有料転換率: 目標 > 20%
  - トライアル開始からの転換日数

継続:
  - MRR（月次定期収益）
  - ARR（年次定期収益）
  - 解約率（月次チャーン）: 目標 < 3%
  - LTV: 目標 > 20,000円
  - NPS（ネットプロモータースコア）

エンゲージメント:
  - DAU/MAU比率: 目標 > 15%（毎日使うユーザー）
  - 週次アクティブユーザー（WAU）
  - 平均セッション数（週次）
  - 機能利用率（QR共有, PDF出力等）

医療・緊急:
  - 緊急公開画面のアクセス数（月次）
  - QRトークン発行数
  - 緊急情報入力完了率（オンボーディング中）

サポート:
  - 問い合わせ件数・カテゴリ
  - 平均解決時間
  - サポート満足度
```

### KPIダッシュボード（管理画面）

```
週次レビュー:
  - KPI前週比較
  - 障害・問い合わせ振り返り

月次レビュー:
  - 解約理由分析
  - コホート分析（登録月別の継続率）
  - 機能利用率の分析

四半期レビュー:
  - 競合比較
  - ロードマップ改訂
  - 価格・機能仮説の検証
```

## レビュー総評（SaaS設計）

### 強み
- 680円という適切な価格設定
- 30日トライアルで価値を体験させる設計
- 医療記録の蓄積によるスイッチングコスト（自然なロックイン）
- Stripe Customer Portal によるセルフサービス

### 課題と改善提案

| 課題 | 改善優先度 | 改善内容 |
|---|---|---|
| ペット死亡時の解約防止 | High | アーカイブモード・思い出機能 |
| 年払いプランがない | High | 月額680円 vs 年6,480円 |
| 紹介プログラムがない | Medium | 紹介コード発行機能 |
| 管理画面が未実装 | High | 商用化前に必須 |
| KPIダッシュボードがない | High | Google Analytics + カスタムダッシュボード |
| エンゲージメント通知が弱い | Medium | プッシュ通知・リマインダー強化 |
| 価格ページが不十分 | Medium | 価値訴求の強化 |
