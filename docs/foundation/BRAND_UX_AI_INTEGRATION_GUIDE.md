# 1. 目的と理念（Overview & Philosophy）

本書は、さけLabにおける「ブランド（Brand）」「体験設計（UX）」「人工知能（AI）」の3軸を横断的に統合し、利用者に一貫した「安心」を提供するための**最上位統合設計ガイドライン**である。

さけLabのAIやUIコンポーネントは、単に効率や美しさを競うために存在するのではない。「安心を設計する」という会社の存在意義を具現化するための手段である。本書は、ブランド感情（Calm / Warmth / Trust）を客観的・定量的に評価し、AI生成コンテンツやUIコピーが常にその品質を満たすためのルールを定める。

# 2. ブランド感情スコア（Brand Emotion Scores）

すべてのUIテキスト、AI提案、デザイン要素は、以下の3つのブランド感情指標によって評価・測定されなければならない。

### 2.1 3大感情スコア定義
1. **Calm Score (落ち着き度)**
   - **定義**: 利用者の焦りや不安を和らげ、客観的かつ冷静な状態を維持させる度合い。
   - **目標値**: 緊急時/医療時 ≥ 90%, 通常時 ≥ 70%
   - **減点対象**: 煽り言葉（「今すぐ！」「大変！」「危険」など感嘆符付き表現）、未確定情報の断定、過度な視覚的強調（極端な赤色や点滅）。
2. **Warmth Score (温かさ・寄り添い度)**
   - **定義**: 機械的な冷たさを排除し、家族やペットの生命を共に大切にするパートナーとしての優しさを感じさせる度合い。
   - **目標値**: 日常/オンボーディング時 ≥ 85%, 緊急時 ≥ 60%
   - **減点対象**: システム中心の表現（「エラーコード: 502」「入力形式エラー」など）、責任転嫁、冷淡な事実のみの羅列。
3. **Trust Score (信頼・誠実度)**
   - **定義**: 提示される情報の正確性、出典、推測と事実の区分、法的/獣医療的な裏付けの透明性を担保する度合い。
   - **目標値**: 全フロー共通 ≥ 95%
   - **減点対象**: 根拠や出典のない記述、曖昧な表現、禁止語（「絶対」「100%」「最強」など）の混入。

### 2.2 スコア算出および評価方法
- **定性チェック**: コピー承認プロセスにおいて、レビューシートに各スコア（0〜100）を自己申告・相互確認により採点し、基準値未満のものは修正対象とする。
- **自動化検証**: 簡易的なテキスト解析により、禁止語の検出および特定のネガティブ・煽り表現の有無をCI環境およびAI生成パイプラインで自動チェックする。

### 2.3 感情スコア評価ルーブリックおよび未達時AI修正プロンプト

#### 【感情スコア評価ルーブリック表 (Evaluation Rubric)】
| スコア軸 | 90〜100点 (理想) | 70〜89点 (許容) | 70点未満 (要修正・不適合) |
| :--- | :--- | :--- | :--- |
| **Calm Score** | 事実と客観的指示のみ。感嘆符ゼロ。焦らせる単語なし。 | 短い注意喚起はあるが煽りはなし。感嘆符1個以下。 | 「今すぐ！」「大変！」「危険」等の煽り・過剰強調あり。 |
| **Warmth Score**| ユーザーやペットの名前に触れ、寄り添う親身なトーン。 | 丁寧語（です・ます）で礼儀正しいが、やや定型文。 | 「エラー502」「入力形式不正」等の冷淡なシステム用語のみ。 |
| **Trust Score** | 根拠・出典・日時・確信度が完備。禁止語ゼロ。 | 出典はあるが根拠データが一部省略されている。 | 禁止語（絶対・100%等）の混入、根拠のない断定。 |

#### 【スコア未達時のAI自動修正プロンプトテンプレート】
```text
[System Prompt for Copy Refactoring]
与えられたUIテキストのブランド感情スコアを修正してください。
Target Criteria: Calm ≥ 90, Warmth ≥ 85, Trust ≥ 95

修正ルール:
1. 感嘆符（！）、誇張表現（大変、危険、急いで）を削除し、事実＋次に行う行動（Who/What/Why）に置き換える。
2. 機械的表現を排除し、「[ペット名]ちゃん」「ご安心ください」等の寄り添い表現を追加する。
3. 「絶対」「100%」等の禁止語が含まれる場合は削除し、根拠（データ・出典）を付記する。
```

# 3. AI出力テンプレートのブランド統合ルール

AI提案や自動生成要約をユーザーに提示する際は、説明責任と情緒的配慮（Warmth & Trust）を両立させるため、以下の統合レイアウトテンプレートを必ず適用する。

### 3.1 統合テンプレート構成要素
1. **要約提案**: Calmかつ簡潔に推奨アクションを提示。
2. **提案の根拠**: なぜその提案をしたのか、ペットの具体的データ（体重、履歴、持病等）を列記。
3. **参照した出典/ガイドライン**: 信頼できる医療資料、ガイドライン名、または元ファイルへのハイパーリンク。
4. **確信度 (Confidence Level)**: 90%以上の場合は「高」、70%〜89%は「中」、それ未満は「低」として表示し、推測の度合いを透明化。
5. **免責事項**: 「AIによる提案であり、直接の獣医師による診断に代わるものではない」旨の標準テキストを常に表示。
6. **更新日時**: 該当データ・提案の作成/最終更新日時（ISO 8601形式）。

### 3.2 UIテキスト表示例
> **🚨 AI提案**: cocoちゃんの呼吸数が通常値より25%上昇しているため、安静状態を保ち、必要に応じて救急病院へ連絡してください。（Calm: 90, Trust: 95）
> - **根拠**: 本日の20:00および22:00の呼吸数測定データ（42回/分、既往歴: 僧帽弁閉鎖不全症）
> - **出典**: さけLab獣医療ガイドライン v1.4 / 日本小動物獣医師会ガイドブック p.12
> - **確信度**: 高 (92%)
> - **更新日時**: 2026-07-22T23:55:00Z
> - **免責**: ※本提案はAIによる自動分析であり、獣医師による診察に代わるものではありません。

### 3.3 例外・不確実性フォールバックUIパターン (Fallback Templates)

AIの確信度が低い場合、データが不足している場合、または情報が対立している場合は、無理な断定を避けて以下のフォールバックテンプレートを必ず適用する。

#### パターン A: 確信度「低」（Confidence < 70%）またはデータ不足時
> **⚠️ AI分析メモ（参考情報）**: 入力された観察データが少ないため、特定の状態を特定できません。
> - **必要な追加アクション**: 呼吸数と体温を計測し、記録を更新してください。
> - **確信度**: 低 (45%)
> - **推奨導線**: 不安な場合は直接 [協力動物病院へ電話] してください。
> - **免責**: ※本表示は確定的な情報ではありません。

#### パターン B: 専門家・ガイドラインの説が複数対立している時
> **💡 AI比較提示**: cocoちゃんの現在の症状（食欲不振・嘔吐）に対して、複数の要因が考えられます。
> - **考えられる要因 1**: 既往歴（胃腸炎）の再発（可能性: 中）
> - **考えられる要因 2**: 処方薬の副作用（可能性: 低）
> - **出典**: さけLab臨床ガイドライン / 獣医内科学テキスト v3
> - **次のステップ**: どちらの可能性においても、まずは絶食・給水観察を推奨します。

# 4. 緊急時コピーの文字数・構造ルール

緊迫した場面（アレルギー発症、怪我、迷子、有事のQRスキャン公開画面など）におけるコピーは、ユーザーのパニックを防ぎ、冷静な判断を支えるため、厳しい制限と構造ルールを課す。

### 4.1 厳しい制約ルール
1. **文字数制限**: 主要な見出しおよび指示コピーは、改行・スペースを含め **最大 60 文字以内** とする。
2. **感嘆符・煽りの禁止**: 感嘆符（「！」、「🚨🚨🚨」の過剰配置）や、不安を煽る言葉（「手遅れになる」「危ない」等）は一切使用しない。
3. **論理的3ステップ構造**:
   - **ステップ1: 対象（Who）** - 誰の、どの状態か。
   - **ステップ2: アクション（What）** - 今何をするか（電話、安静、搬送等）。
   - **ステップ3: 理由（Why）** - なぜそれが必要か（持病、薬の影響等）。

### 4.2 コピー構造化例
- **正しい例（52文字）**:
  「cocoちゃんが心臓疾患薬を誤飲した可能性があります。直ちに安静にし、以下の緊急獣医師へ連絡してください。」
- **アンチパターン（煽り/長文）**:
  「大変です！危険な薬の誤飲が発生しました！命に関わるので、今すぐ急いで緊急夜間救急病院に電話をしてください！！早くしないと手遅れになります！」

### 4.3 緊急コピー文字数カウント標準規約 (Char Count Specification)

1. **カウント算出式**:
   - 日本語・全角文字（漢字・ひらがな・カタカナ・全角記号）: **1文字 = 1カウント**
   - 半角英数字・半角スペース・半角記号: **0.5文字 = 0.5カウント**
   - 改行コード (`\n` / `\r\n`): **カウント対象外** (0カウント)
   - 絵文字 (Unicode Emoji): **1絵文字 = 2カウント** (過剰使用防止のため)
2. **言語別上限値規約**:
   - **日本語 (ja)**: 最大 **60 文字**
   - **英語 (en)**: 最大 **160 文字 (160 Characters equivalent)**
   - **多言語拡張時**: 日本語をマスターとし、画面幅 320px（モバイル最小幅）で3行以内に収まるテキスト長を絶対上限とする。

# 5. コピー承認・AI監査の統合ルール

ブランドの品質と法的/ガバナンス要件（DESIGN_GOVERNANCE.md）を接続するための承認・監査運用を定義する。

### 5.1 コピー承認プロセス（ブランド統合チェック）
- コピー承認申請テンプレート（`BRAND_GUIDELINES.md` に定義）に、以下の評価欄を必須追記する。
  - `Calm Score: [算出スコア/目標値]`
  - `Warmth Score: [算出スコア/目標値]`
  - `Trust Score: [算出スコア/目標値]`
  - `AIブランド整合チェック: [合格/不適合]`（AIによる文章出力の場合のみ適用）
- 承認フローは、作成者 → マーケティング責任者 → Brand Owner（最終承認）とし、Brand Integration Lead の関与も推奨する。

### 5.2 AI監査ログへのブランドタグ付与
- `DESIGN_GOVERNANCE.md` に定める AI メタデータおよびログ保存において、監査ログに以下のブランド整合性に関するメタデータ項目を追加保存することを必須化する。
  ```json
  "brand_evaluation": {
    "calm_score": 92,
    "warmth_score": 75,
    "trust_score": 98,
    "brand_checklist_passed": true,
    "brand_tags": ["calm_emergency", "no_prohibited_words"]
  }
  ```

# 6. 全ドキュメントの横断整合性チェックリスト（Alignment Checklist）

デザイン基盤に関わるすべての改訂や開発時に、本ガイドラインが遵守されているかを確認するチェックリスト。

- [ ] **1. 感情スコア基準**: Calm/Warmth/Trust の各基準値が、作成したUI・コピー・AI出力において担保されているか。
- [ ] **2. 禁止語チェック**: `BRAND_GUIDELINES.md` に定める「禁止語リスト（最強、革命、絶対等）」が完全に排除されているか。
- [ ] **3. AI説明責任**: AI提案時に「要約」「根拠」「出典」「確信度」「免責」「更新日時」が揃っているか。
- [ ] **4. 緊急時60文字**: 緊急コピーが60文字以内で、感嘆符がなく、論理的3ステップ構造を満たしているか。
- [ ] **5. 監査タグ紐づけ**: 保存されるAI提案のメタデータに `brand_evaluation` タグが定義されているか。
- [ ] **6. 相互参照リンク**: ドキュメント間で相互のアンカー（例: `BRAND_UX_AI_INTEGRATION_GUIDE.md #4`）が定義され、分断がないか。
