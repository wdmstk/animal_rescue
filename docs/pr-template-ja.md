## Pull Request 日本語テンプレート

### 概要
- 何を実装したかを1-2行で記載
- 関連Issue番号を記載（例: `#123`）
- 並行実装用にどうIssue分割したかを1行で記載（API/UI/Test/Docsなど）

### 変更内容
- 主要変更を箇条書き

### テスト結果
- `npm run lint` : 成功/失敗
- `npx vitest run` : 成功/失敗
- `npm run test:e2e` : 成功/失敗（該当時）

### 確認観点・未対応
- レビューで見てほしいポイント
- 後続タスクに回す項目
- TASK完了時に対応Issueをクローズすることを確認
