## Pull Request 日本語テンプレート

### 概要
- 何を実装したかを1-2行で記載
- 関連Issue番号を記載（例: `#123`）
- 並行実装用にどうIssue分割したかを1行で記載（API/UI/Test/Docsなど）

### 変更内容
- 主要変更を箇条書き

### テスト結果
- `npm run lint` : 成功/失敗
- `npx vitest run` : 成功/失敗
- `npm run test:e2e` : 成功/失敗（該当時）

### 確認観点・未対応
- レビューで見てほしいポイント
- 後続タスクに回す項目
- PR作成前に一次セルフレビュー（意図した差分・不要変更混入の有無）を実施
- PRは通常PR（Ready for review）で作成し、ドラフトの場合は理由を記載
- CI必須チェック完了後に最終セルフレビューを実施したことを確認
- 最終セルフレビューはPR本文に次の3観点を必ず記載
  - 差分妥当性:
  - 不要変更混入:
  - 残リスク/フォローアップ:
- マージ後、TASK完了時に対応Issueをクローズしたことを確認
- チェックリストはCIで段階的に検証される（PR作成前/作成時、マージ前、マージ後）
- PR本文編集（`edited`）でガードチェックが再評価されるため、チェック更新のための空コミットは不要
